# Testing the Automated Pipeline

## Quick Test Schedule

Updated cron schedules for immediate testing:
- **check-new-filings**: Every 5 minutes
- **submit-batch**: Every 5 minutes
- **poll-batches**: Every 2 minutes

## Option 1: Manual Trigger (Immediate Testing)

You can manually trigger each endpoint without waiting for the cron schedule.

### Step 1: Get Your CRON_SECRET

After deploying to Vercel:
1. Go to Vercel Dashboard → Your Project → Settings → Environment Variables
2. Find `CRON_SECRET` (auto-generated by Vercel)
3. Copy the value

### Step 2: Trigger Each Endpoint

```bash
# Set your secret (get from Vercel dashboard)
export CRON_SECRET="your_secret_here"
export DOMAIN="your-domain.vercel.app"

# 1. Check for new filings (finds 424B4s from SEC)
curl -X GET "https://$DOMAIN/api/cron/check-new-filings" \
  -H "Authorization: Bearer $CRON_SECRET"

# Expected output:
# {
#   "success": true,
#   "checked": 100,
#   "newFilings": 5,
#   "filings": ["ServiceTitan, Inc. (0001234567-24-000123)", ...],
#   "timestamp": "2025-12-20T14:00:00Z"
# }

# 2. Submit batch (download PDFs, upload to blob, send to Claude)
curl -X GET "https://$DOMAIN/api/cron/submit-batch" \
  -H "Authorization: Bearer $CRON_SECRET"

# Expected output:
# {
#   "success": true,
#   "processed": 3,
#   "batchId": "msgbatch_01ABC123",
#   "filingCount": 3,
#   "status": "in_progress",
#   "expiresAt": "2025-12-21T14:00:00Z"
# }

# 3. Poll for batch results (check if Claude finished)
curl -X GET "https://$DOMAIN/api/cron/poll-batches" \
  -H "Authorization: Bearer $CRON_SECRET"

# Expected output:
# {
#   "success": true,
#   "batches": [
#     {
#       "batchId": "msgbatch_01ABC123",
#       "status": "ended",
#       "succeeded": 3,
#       "failed": 0
#     }
#   ]
# }

# 4. Check overall status
curl "https://$DOMAIN/api/cron/status"

# Expected output:
# {
#   "health": { "status": "healthy", "successRate": "100%" },
#   "filings": { "total": 5, "pending": 0, "processing": 0, "completed": 5 },
#   "batches": { "active": 0, "completed": 1 },
#   "recentFilings": [...]
# }
```

**Go to Neon Dashboard:**
1. Open your Neon project
2. Go to SQL Editor
3. Run these queries:

```sql
-- Check filings table
SELECT
  ticker,
  filing_date,
  status,
  blob_url,
  batch_id,
  created_at,
  processed_at
FROM filings
ORDER BY created_at DESC
LIMIT 10;

-- Check metrics extracted
SELECT
  f.ticker,
  m.metric_name,
  m.metric_value,
  m.page_number,
  m.confidence
FROM metrics m
JOIN filings f ON m.filing_id = f.id
ORDER BY m.created_at DESC
LIMIT 20;

-- Count metrics per filing
SELECT
  f.ticker,
  COUNT(m.id) as metric_count,
  f.status,
  f.processed_at
FROM filings f
LEFT JOIN metrics m ON m.filing_id = f.id
GROUP BY f.id, f.ticker, f.status, f.processed_at
ORDER BY f.processed_at DESC;

-- Check batch jobs
SELECT
  batch_id,
  status,
  filing_count,
  submitted_at,
  completed_at,
  request_counts
FROM batch_jobs
ORDER BY submitted_at DESC
LIMIT 5;
```

## Troubleshooting

### No New Filings Found

**Problem**: `check-new-filings` returns `newFilings: 0`

**Possible Causes:**
- All current 424B4s are already in database
- SEC RSS feed temporarily unavailable

**Solution:**
```sql
-- Clear database to re-process filings
DELETE FROM metrics;
DELETE FROM filings;

-- Try again
curl -X GET "https://$DOMAIN/api/cron/check-new-filings" \
  -H "Authorization: Bearer $CRON_SECRET"
```

### Batch Submission Fails

**Problem**: `submit-batch` returns error or `processed: 0`

**Possible Causes:**
- No pending filings in database
- SEC download blocked (User-Agent issue)
- Blob storage quota exceeded

**Solution:**
```bash
# Check Vercel logs for specific error
# Verify SEC_USER_AGENT is set correctly
# Format: "YourCompany support@email.com"

# Check pending filings
curl "https://$DOMAIN/api/cron/status" | jq '.filings'
```

### Batch Stuck in Processing

**Problem**: Batch stays "in_progress" for >2 hours

**Possible Causes:**
- Claude Batch API processing time varies (can take 30min-2hr)
- Batch failed or expired

**Solution:**
```bash
# Check batch status
curl -X GET "https://$DOMAIN/api/cron/poll-batches" \
  -H "Authorization: Bearer $CRON_SECRET" | jq

# If batch is stuck, check Claude dashboard
# https://console.anthropic.com/settings/usage
```

### No Metrics in Database

**Problem**: Filing is "completed" but no metrics

**Possible Causes:**
- Claude extraction failed
- Response parsing error
- Database insertion error

**Solution:**
```sql
-- Check for error messages
SELECT ticker, status, error_message, retry_count
FROM filings
WHERE status = 'failed'
ORDER BY updated_at DESC;

-- Check Vercel logs for parsing errors
```

## Resetting for Fresh Test

If you want to start over:

```sql
-- Clear all data
DELETE FROM metrics;
DELETE FROM batch_jobs;
DELETE FROM filings;
DELETE FROM pipeline_state;

-- Reset pipeline state
INSERT INTO pipeline_state (id, total_processed)
VALUES (1, 0);
```

Then trigger the pipeline again.

## Success Criteria

✅ **Step 1 Complete** when:
- `check-new-filings` returns `newFilings > 0`
- Neon `filings` table has new rows with `status='pending'`

✅ **Step 2 Complete** when:
- `submit-batch` returns `batchId`
- Vercel Blob has PDFs in `filings/424b4/` folder
- Neon `filings` table shows `status='processing'` and `blob_url` is set

✅ **Step 3 Complete** when:
- `poll-batches` returns `status='ended'` and `succeeded > 0`
- Neon `metrics` table has rows for each filing
- Neon `filings` table shows `status='completed'`

✅ **Full Pipeline Success** when:
- Frontend shows the new company with all metrics
- Source panel opens PDF at correct page
- Status dashboard shows "healthy"

## After Testing

Once you've verified the pipeline works, reset to production schedule:

```json
// vercel.json
"crons": [
  {
    "path": "/api/cron/check-new-filings",
    "schedule": "0 * * * *"  // Every hour
  },
  {
    "path": "/api/cron/submit-batch",
    "schedule": "15 */2 * * *"  // Every 2 hours at :15
  },
  {
    "path": "/api/cron/poll-batches",
    "schedule": "30,45 * * * *"  // Twice per hour
  }
]
```

Then redeploy:
```bash
git add vercel.json
git commit -m "chore: restore production cron schedule"
vercel --prod
```
